## 🎭 What are Playwright Test Agents?

Playwright Test Agents are AI-powered tools that help you create, generate, and maintain automated tests:

- **🎯 Planner Agent**: Explores your application and creates comprehensive test plans
- **🔧 Generator Agent**: Transforms test plans into executable Playwright tests
- **🩹 Healer Agent**: Automatically fixes failing tests when the UI changes

## Installation

1. **Clone or navigate to this project:**

   ```bash
   cd saucedemo-playwright-agents
   ```

2. **Install dependencies:**

   ```bash
   npm install
   ```

3. **Install Playwright browsers:**

   ```bash
   npx playwright install
   ```

4. **Verify installation:**
   ```bash
   npx playwright test tests/seed.spec.ts
   ```

## Using the Agents

### Planner Agent

The Planner Agent explores your application and creates detailed test plans.

#### How to Use:

1. **Open GitHub Copilot Chat** in VS Code
2. **Tag the Planner Agent**: `@playwright-test-planner`
3. **Provide instructions**, for example:

```
@playwright-test-planner

Explore the SauceDemo shopping cart functionality and create a comprehensive test plan.
Focus on:
- Adding items to cart
- Removing items from cart
- Cart persistence
- Checkout flow

Use tests/seed.spec.ts as the seed file.
```

#### What the Planner Does:

1. Sets up a browser session
2. Navigates through the application
3. Identifies user flows and interactions
4. Creates detailed test scenarios
5. Saves the plan to `specs/` directory as Markdown

#### Example Output:

The Planner will create a file like `specs/shopping-cart.md`:

```markdown
# Shopping Cart Tests

**Seed:** `tests/seed.spec.ts`

## 1. Adding Items to Cart

### 1.1 Add Single Item

**Steps:**

1. Click "Add to cart" button for Sauce Labs Backpack
2. Verify cart badge shows "1"
3. Click shopping cart icon
4. Verify item appears in cart

**Expected:** Item is added successfully

### 1.2 Add Multiple Items

...
```

### Generator Agent

The Generator Agent converts test plans into executable Playwright tests.

#### How to Use:

1. **Ensure you have a test plan** in `specs/` directory
2. **Open GitHub Copilot Chat** in VS Code
3. **Tag the Generator Agent**: `@playwright-test-generator`
4. **Provide instructions**, for example:

```
@playwright-test-generator

Generate tests for the shopping cart test plan in specs/shopping-cart.md
Create test file: tests/shopping-cart.spec.ts
```

#### What the Generator Does:

1. Reads the test plan from `specs/`
2. Sets up a browser session
3. Executes each step manually to verify it works
4. Generates Playwright test code
5. Saves the test file to `tests/` directory

#### Example Output:

The Generator creates `tests/shopping-cart.spec.ts`:

```typescript
// spec: specs/shopping-cart.md
// seed: tests/seed.spec.ts

import { test, expect } from "@playwright/test";

test.describe("Shopping Cart Tests", () => {
  test("Add Single Item", async ({ page }) => {
    // Navigate and login (from seed)
    await page.goto("https://www.saucedemo.com/");
    await page.fill('[data-test="username"]', "standard_user");
    await page.fill('[data-test="password"]', "secret_sauce");
    await page.click('[data-test="login-button"]');

    // 1. Click "Add to cart" button for Sauce Labs Backpack
    await page.click('[data-test="add-to-cart-sauce-labs-backpack"]');

    // 2. Verify cart badge shows "1"
    await expect(page.locator(".shopping_cart_badge")).toHaveText("1");

    // 3. Click shopping cart icon
    await page.click(".shopping_cart_link");

    // 4. Verify item appears in cart
    await expect(page.locator(".cart_item")).toBeVisible();
  });
});
```

### Healer Agent

The Healer Agent automatically fixes failing tests.

#### How to Use:

1. **Run your tests** and identify failures:

   ```bash
   npx playwright test
   ```

2. **Open GitHub Copilot Chat** in VS Code
3. **Tag the Healer Agent**: `@playwright-test-healer`
4. **Provide the failing test**, for example:

```
@playwright-test-healer

Fix the failing test: tests/shopping-cart.spec.ts
Test name: "Add Single Item"
```

#### What the Healer Does:

1. Analyzes the test failure
2. Runs the test to identify the issue
3. Suggests fixes (locator updates, waits, data changes)
4. Applies patches to the test
5. Re-runs until the test passes or reaches guardrails

#### Common Fixes:

- **Locator updates**: When element selectors change
- **Wait adjustments**: When timing issues occur
- **Data fixes**: When test data becomes invalid
- **Flow corrections**: When user flows change

## CI/CD Integration

### GitHub Actions

A workflow file is included at `.github/workflows/copilot-setup-steps.yml`.

To set up continuous testing:

1. **Push your code to GitHub**
2. **Enable GitHub Actions** in your repository
3. **Tests will run automatically** on push and pull requests

### Custom CI/CD

Add this to your CI pipeline:

```yaml
- name: Install dependencies
  run: npm ci

- name: Install Playwright browsers
  run: npx playwright install --with-deps

- name: Run tests
  run: npx playwright test

- name: Upload test results
  if: always()
  uses: actions/upload-artifact@v3
  with:
    name: playwright-report
    path: playwright-report/
```

## Best Practices

### 1. Start with the Seed Test

Always create a seed test that demonstrates the basic functionality of your application. This helps the agents understand your app's structure.

### 2. Be Specific with Planner Instructions

When using the Planner Agent, provide clear, specific instructions about what you want to test:

✅ **Good:**

```
Create a test plan for the checkout flow, including:
- Form validation
- Payment processing
- Order confirmation
```

❌ **Bad:**

```
Test the checkout
```

### 3. Review Generated Tests

Always review the tests generated by the Generator Agent before committing them. The agents are powerful but may need human oversight.

### 4. Use Healer for Maintenance

When UI changes break tests, use the Healer Agent instead of manually fixing them. It's faster and learns from patterns.

### 5. Organize Test Plans

Keep your test plans organized in the `specs/` directory:

```
specs/
├── authentication.md
├── shopping-cart.md
├── checkout-flow.md
└── product-catalog.md
```

### 6. Keep Tests Independent

Each test should be able to run independently. Don't rely on test execution order.

### 7. Use Descriptive Names

Use clear, descriptive names for test files and test cases:

```typescript
// Good
test('should add item to cart and update badge count', ...)

// Bad
test('test1', ...)
```

## Example Workflow

Here's a complete workflow for creating tests with agents:

### Step 1: Create Test Plan

```
@playwright-test-planner

Create a comprehensive test plan for SauceDemo product filtering and sorting.
Include:
- Filter by price (low to high, high to low)
- Filter by name (A-Z, Z-A)
- Verify product order after each filter
- Test with different user types

Seed: tests/seed.spec.ts
```

### Step 2: Generate Tests

```
@playwright-test-generator

Generate tests from specs/product-filtering.md
Output file: tests/product-filtering.spec.ts
```

### Step 3: Run Tests

```bash
npx playwright test tests/product-filtering.spec.ts
```

### Step 4: Fix Failures (if any)

```
@playwright-test-healer

Fix failing test in tests/product-filtering.spec.ts
Test: "Filter by price low to high"
```

### Step 5: Review and Commit

```bash
git add .
git commit -m "Add product filtering tests"
git push
```

## Troubleshooting

### Agents Not Responding

1. Ensure GitHub Copilot is enabled in VS Code
2. Check that MCP configuration is correct in `.vscode/mcp.json`
3. Restart VS Code

### Tests Failing

1. Run in headed mode to see what's happening: `npx playwright test --headed`
2. Use debug mode: `npx playwright test --debug`
3. Check if SauceDemo is accessible
4. Try the Healer Agent to auto-fix

### Browser Installation Issues

```bash
# Reinstall browsers
npx playwright install --force

# Install with system dependencies
npx playwright install --with-deps
```
